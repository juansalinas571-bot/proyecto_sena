01 · Juan · 5 minutos

¿Qué hicimos y por qué?

Contexto, tecnologías y organización del proyecto
"Hola a todos, somos Juan, Ehidan y Krispi. Hoy venimos a presentarles Secure Access, un sistema que hicimos en el SENA para controlar el acceso a un lugar usando reconocimiento facial."

El problema de la vida real:

Pensemos en un edificio, una bodega o una oficina donde necesita haber un vigilante revisando documentos uno por uno. Eso es demorado, la gente se cansa, se pueden equivocar y lo peor: no queda ningún registro de quién entró, a qué hora, o si alguien intentó entrar sin permiso.

¿Qué hicimos para solucionarlo?

Desarrollamos una aplicación web donde:

  • Registramos a las personas con su foto, nombre, documento y una contraseña
  • Cuando quieren entrar, solo se ponen frente a la cámara y el sistema los reconoce
  • Si la cámara falla, pueden escribir su documento y contraseña como respaldo
  • Todo queda guardado: quién entró, a qué hora, si usó cámara o contraseña, desde qué computador

Tecnologías que usamos

PHP 8.2 con CodeIgniter 4 para el servidor, MySQL para la base de datos.

Del lado del navegador

JavaScript puro para manejar la cámara y el reconocimiento facial, y CSS con colores oscuros y neón para una interfaz moderna.

¿Y cómo está organizado el proyecto?

Usamos el patrón MVC (Modelo-Vista-Controlador), que básicamente es separar la lógica en tres partes: los Controladores reciben las peticiones y deciden qué hacer, los Modelos se encargan de hablar con la base de datos, y las Vistas son el HTML y JavaScript que ve el usuario en pantalla.

La base de datos tiene 4 tablas:

  • users: las personas que usan el sistema (admin, supervisor, vigilante)
  • faces: los rostros registrados con nombre, documento y contraseña
  • access_logs: el historial de cada intento de entrada (quién, cuándo, cómo, si entró o no)
  • migrations: el control de versiones de la base de datos

02 · Ehidan · 5 minutos

¿Cómo funciona el sistema por dentro?

Rutas, seguridad, reconocimiento facial y reportes
"Ahora les voy a explicar cómo se conecta todo esto internamente. Cómo hicimos para que cuando alguien hace clic en un botón, pase lo que tiene que pasar."

Las rutas: el mapa del sistema

En el archivo Routes.php definimos todas las direcciones (rutas) del sistema. Es como un mapa que le dice al servidor qué hacer cuando alguien pide una página.

/login → formulario de inicio de sesión /home → panel principal con estadísticas /faces → registrar y ver rostros /camera → control de acceso con cámara /reports → historial de accesos /users → gestión de usuarios del sistema

Los guardias de seguridad (Filters)

Tenemos filtros que revisan si alguien tiene permiso para entrar. El principal es el AuthFilter: revisa que hayas iniciado sesión en el navegador. Si no, te manda al login automáticamente.

¿Y cómo funciona lo del reconocimiento facial?

Usamos algo que se llama dHash (Difference Hash). Es una forma de convertir una foto en un código de 64 dígitos. Cuando alguien se pone frente a la cámara:

  • El navegador toma la foto y la reduce a 9x8 píxeles en blanco y negro
  • Compara cada píxel con el que tiene al lado y genera un código único de 64 bits
  • Ese código se manda al servidor para compararlo con los códigos guardados
  • Si dos códigos se parecen mucho (tienen pocas diferencias), es la misma persona

Lo bueno es que no necesitamos inteligencia artificial ni servicios de pago. Todo se hace con JavaScript puro en el navegador.

Límite de intentos

Si alguien falla 3 veces seguidas con la cámara, el sistema le pide que ponga su documento y contraseña. Esto evita que estén probando fotos una y otra vez.

Los reportes

Todos los intentos de acceso quedan guardados en la base de datos con IP, fecha, estado (éxito o denegado) y método (cámara o contraseña). Después se pueden consultar en la sección Reportes, filtrando por fecha, por estado o buscando por nombre. Muestra 25 resultados por página para no saturar.

03 · Krispi · 5 minutos

Veámoslo en vivo

Demostración paso a paso y cierre
"Bueno, ya les explicaron de qué se trata y cómo funciona por dentro. Ahora vamos a mostrar que todo esto sí sirve. Préndamoslo y veámoslo en acción."

Paso 1: Iniciar sesión

Abro el navegador y voy a la página. Ingreso con admin / admin123. Aparece el dashboard con las estadísticas: cuántos rostros hay registrados, cuántos usuarios activos y los últimos registros.

Paso 2: Registrar una persona

Voy a "Registro de Rostros". Lleno el nombre, número de documento y una contraseña. Puedo subir una foto desde el computador o tomarla con la cámara web. Le doy guardar y ya queda registrado en la base de datos.

Paso 3: Control de acceso

Voy a "Control de Acceso". Selecciono la persona que quiero verificar, me pongo frente a la cámara y el sistema me reconoce. Cada intento queda registrado. Si fallo 3 veces, me aparece un formulario para poner documento y contraseña.

Si la cámara funciona

Sale un modal verde con acceso exitoso. La persona queda registrada en el historial.

Si falla

Sale un modal rojo de acceso denegado. Después de 3 intentos, pide documento y contraseña.

Paso 4: Reportes

Voy a "Reportes" y puedo ver las estadísticas del día: cuántos accesos exitosos, cuántos denegados, y el total acumulado. Puedo filtrar por fecha, por estado o buscar a alguien por nombre o documento. La tabla se va pasando de página en página de a 25 registros.

Paso 5: Seguridad

  • Si cierro el navegador y vuelvo a entrar, me pide login otra vez
  • Un usuario con rol vigilante no puede crear ni eliminar rostros, solo verlos
  • El sistema no deja borrar tu propio usuario para que siempre haya al menos un administrador
Importante para la demo: Si la cámara no funciona o da problemas, no hay problema, usamos el método de documento + contraseña que siempre sirve.

Para cerrar:

"Con este proyecto demostramos que se puede construir un sistema de control de acceso funcional usando solo tecnologías web que todos conocemos: PHP, JavaScript y MySQL. No necesitamos comprar equipos especiales ni pagar inteligencia artificial. El código está comentado línea por línea y la interfaz es moderna con colores neón oscuros."
