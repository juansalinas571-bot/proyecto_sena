POSTMAN · PASO A PASO

Aprende a hacer un CRUD completo de usuarios en Secure Access

Vas a hacer 5 operaciones en orden: CREAR → CONSULTAR → ACTUALIZAR → CONSULTAR → ELIMINAR → CONSULTAR

1

Iniciar sesión (Login)

Esto se hace una sola vez. El token dura 24 horas.

POST http://localhost/proyecto_sena/api/auth/login
HEADERS
Content-Type: application/json
BODY → raw → JSON
{
"username": "admin",
"password": "admin123"
}
RESPUESTA (si funciona)
{
"status": "success",
"data": {
"token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
"user": { "username": "admin", "role": "admin" }
}
}
IMPORTANTE: Copia el token (el texto largo que empieza con eyJ0...) y pégalo en un bloc de notas. Lo vas a usar en todos los pasos siguientes.
2

CREAR un usuario

Crea un usuario nuevo en el sistema. Solo administradores.

POST http://localhost/proyecto_sena/api/usuarios/crear
HEADERS
Content-Type: application/json
Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9... (el token que copiaste)
BODY → raw → JSON
{
"username": "miusuario",
"password": "123456",
"full_name": "Mi Nombre",
"role": "vigilante"
}
RESPUESTA (si funciona, código 201)
{
"status": "success",
"message": "Usuario creado exitosamente",
"data": { "id": 3, "username": "miusuario", ... }
}
Roles disponibles: admin, supervisor, vigilante. Si no pones rol, por defecto queda "vigilante".
3

CONSULTAR los usuarios

Dos formas: ver todos o ver uno solo.

GET http://localhost/proyecto_sena/api/usuarios
HEADERS (solo uno)
Authorization: Bearer tu_token
RESPUESTA
{
"status": "success",
"data": {
"users": [ { "id": 1, "username": "admin", ... }, { "id": 3, "username": "miusuario", ... } ],
"total": 3, "page": 1, "limit": 10
}
}

Para ver un solo usuario: Cambia el método a GET y la URL a http://localhost/proyecto_sena/api/usuarios/3 (donde 3 es el ID del usuario que quieres ver).

4

ACTUALIZAR un usuario

Modifica los datos de un usuario existente. Cambia solo los campos que quieras.

PUT http://localhost/proyecto_sena/api/usuarios/actualizar/3
HEADERS
Content-Type: application/json
Authorization: Bearer tu_token
BODY → raw → JSON
{
"full_name": "Nombre Cambiado",
"role": "supervisor"
}
RESPUESTA
{
"status": "success",
"message": "Usuario actualizado exitosamente",
"data": { "id": 3, "username": "miusuario", "full_name": "Nombre Cambiado", "role": "supervisor" }
}
OJO: El método debe ser PUT, no GET. Si usas GET te va a dar error 404. Arriba al lado de la URL donde dice "GET" haz clic y cámbialo a "PUT".
5

Verificar el cambio

Vuelve a consultar para confirmar que el usuario se actualizó.

GET http://localhost/proyecto_sena/api/usuarios/3
HEADERS
Authorization: Bearer tu_token
RESPUESTA
{
"status": "success",
"data": { "id": 3, "username": "miusuario", "full_name": "Nombre Cambiado", "role": "supervisor" }
}

Si ves el nombre y el rol actualizados, todo salió bien. Si no, revisa que el método sea GET y el ID correcto.

6

ELIMINAR un usuario

Borra un usuario del sistema. No te puedes eliminar a ti mismo.

DELETE http://localhost/proyecto_sena/api/usuarios/eliminar/3
HEADERS (solo uno)
Authorization: Bearer tu_token
RESPUESTA
{
"status": "success",
"message": "Usuario eliminado exitosamente"
}
7

Confirmar que se eliminó

Consulta la lista otra vez. El usuario borrado ya no debe aparecer.

GET http://localhost/proyecto_sena/api/usuarios
HEADERS
Authorization: Bearer tu_token
RESPUESTA (el usuario con ID 3 ya no aparece)
{
"status": "success",
"data": {
"users": [ { "id": 1, "username": "admin" }, { "id": 2, "username": "operador1" } ],
"total": 2
}
}

Resumen rápido

Operación Método URL Headers
Login POST /api/auth/login Content-Type
CREAR POST /api/usuarios/crear Content-Type + Bearer
CONSULTAR GET /api/usuarios Bearer
CONSULTAR 1 GET /api/usuarios/3 Bearer
ACTUALIZAR PUT /api/usuarios/actualizar/3 Content-Type + Bearer
ELIMINAR DELETE /api/usuarios/eliminar/3 Bearer

Errores comunes y cómo solucionarlos

Error Qué significa Solución
401 Token inválido El token no es correcto o no lo enviaste Revisa el header Authorization. Haz Login otra vez
401 Token no proporcionado Falta el header Authorization Agrega: Authorization: Bearer tu_token
404 Route not found La URL o el método no existen Revisa que escribiste bien la URL y el método (GET, POST, PUT, DELETE)
404 GET: api/usuarios/actualizar/2 Usaste GET en vez de PUT Cambia el método a PUT
422 Error de validación Faltan campos obligatorios Revisa que username, password y full_name no estén vacíos
¿Todo falla? Borra la colección de Postman, vuelve a importar Secure-Access-Postman.json y empieza de nuevo desde el Login. A veces las variables se dañan.
