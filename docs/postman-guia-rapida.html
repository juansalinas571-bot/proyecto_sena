CRUD USUARIOS

Aprende a Crear, Consultar, Actualizar y Eliminar usuarios desde Postman

Esta guía te explica paso a paso cómo hacer un CRUD completo de usuarios usando la API de Secure Access desde Postman.
Las operaciones están separadas en 4 archivos en app/Controllers/Api/: Crear.php, Consultar.php, Actualizar.php y Eliminar.php. Cada uno hace exactamente lo que su nombre dice.

+

Crear usuario

POST /api/usuarios/crear

Crea un nuevo usuario en el sistema. Solo administradores.

{ "username": "operador1", "password": "secreto123", "full_name": "Operador Uno", "role": "vigilante" }
☷

Consultar usuarios

GET /api/usuarios

Lista todos los usuarios. También puedes ver uno solo:

GET /api/usuarios/2

Solo administradores.

✎

Actualizar usuario

PUT /api/usuarios/actualizar/2

Actualiza los datos de un usuario existente. Solo administradores.

{ "full_name": "Operador Modificado", "role": "supervisor" }
✕

Eliminar usuario

DELETE /api/usuarios/eliminar/2

Elimina un usuario del sistema. No puedes eliminarte a ti mismo.

Solo administradores.

1 Importar la colección en Postman

  1. Abre Postman y haz clic en Import (arriba a la izquierda)
  2. Selecciona el archivo docs/Secure-Access-Postman.json
  3. Listo, ya tienes la colección Secure Access API con todas las carpetas
Las variables como {{base_url}} y {{token}} ya vienen configuradas. No tienes que preocuparte por eso.

2 Iniciar sesión (Login)

  1. En la colección, abre la carpeta 1. Auth y selecciona Login
  2. Haz clic en Send
  3. Automáticamente el token se guarda en la variable {{token}}
  4. Ya puedes usar todos los demás endpoints
// Respuesta del Login
{
"status": "success",
"data": {
"token": "eyJ0eXAiOiJKV1Qi...",
"user": { "username": "admin", "role": "admin" }
}
}

3 Probar el CRUD de usuarios

  1. CREAR: Ve a 2. Usuarios (CRUD) → CREAR usuario. Haz Send. Debe responder código 201.
  2. CONSULTAR: Ve a CONSULTAR todos los usuarios. Haz Send. Verás la lista completa.
  3. CONSULTAR uno: Ve a CONSULTAR usuario por ID. Cambia el número si quieres.
  4. ACTUALIZAR: Ve a ACTUALIZAR usuario. El body ya viene con datos de ejemplo.
  5. ELIMINAR: Ve a ELIMINAR usuario. Cuidado: esta operación no se puede deshacer.
Después de crear un usuario, ve a CONSULTAR para verificar que apareció. Después de actualizarlo, vuelve a consultar para ver los cambios.

Probar la seguridad

¿Qué pasa si alguien intenta acceder sin autorización? La API responde con código 401.

// Sin token o con token inválido
{
"status": "error",
"message": "No autorizado"
}

En la carpeta 4. Pruebas de Seguridad de la colección tienes dos ejemplos listos para probar.

✋ Tips para la expo

  1. Orden sugerido: Login → CREAR → CONSULTAR (ver el nuevo) → ACTUALIZAR → CONSULTAR (ver cambios) → ELIMINAR → CONSULTAR (ya no aparece)
  2. Si el token expiró (24h), solo vuelve a ejecutar Login y se renueva solo
  3. Para la seguridad: muestra primero el CONSULTAR funcionando y luego el "Sin token" para que se vea la diferencia
  4. Los datos de ejemplo vienen con username: admin y password: admin123
